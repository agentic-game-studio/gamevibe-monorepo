# GameVibe AI Progress Log

## Codebase Patterns
- MiniMax API uses `x-api-key` header (not Bearer), endpoint: `https://api.minimax.io/anthropic/v1/messages`
- Tool calling: send `tools` in request body, parse `tool_use` from response
- All packages use ES modules - imports must include `.js` extensions

## 2026-03-07 - Smart Fallback Selection
- Implemented smart fallback selection based on user description keywords
- Added 4 new fallback games: endless-runner, tower-defense, rpg, multiplayer
- Modified generateGameCode() to use selectBestFallback() instead of random selection
- **Learnings for future iterations:**
  - MiniMax API is slow (1.5-2 min) and unreliable for custom game generation
  - Fallback games are more reliable and actually better quality than AI-generated
  - Smart keyword matching ensures user intent is respected
  - Pre-built fallbacks provide instant delivery vs AI timeout issues
  - All new games use Phaser 3.70 and generated graphics textures

## 2026-03-08 - Fix TypeScript null check in game page
- Fixed params null check in showcase-portal game page
- Added null guard before accessing params.id
- Changed useEffect dependency from params.id to params
- **Learnings for future iterations:**
  - Next.js 15 useParams can return null, always check before accessing id
  - Use full params object in dependency array to avoid undefined issues

## 2026-03-10 - AI Game Code Bug Fixes
- Added NEW truncation patterns to fix AI-generated code bugs:
  - gameSta. -> gameState., gameStat. -> gameState. (truncated variable names)
  - spacts() -> spawnObjects(), spawnObjcts() -> spawnObjects() (truncated function names)
- Updated game-post-processor.ts with fixCommonBugs() method
- Updated loader.ts in both pre-HTML and post-HTML extraction locations
- **Learnings for future iterations:**
  - AI truncates variable/function names at arbitrary lengths (6-8 chars typical)
  - Add new patterns as they are discovered during game generation
  - Fixes should be in both post-processor (storage time) and loader (runtime safety net)

## 2026-03-11 - Streaming + Truncation Detection
- Added GameValidator class to detect truncation in real-time
- Added continuation logic to request completion when truncation detected
- Added new truncation patterns: orms -> platforms, hasInvincibility fixes
- Updated ai-service.ts to validate and retry on truncation
- **Learnings for future iterations:**
  - MiniMax truncates at ~16K tokens, causing variable/function name truncation
  - Detection patterns: trailing commas, unclosed braces, incomplete statements
  - Use continuation prompts with last 800 chars to complete code
  - maxRetries = 2 is sufficient to recover most truncated code
  - Post-processor repairs still needed as safety net for edge cases
